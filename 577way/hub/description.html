VGP 577-way PhyloP Conservation

Per-base PhyloP conservation tracks computed from vgp-577way-v1, a Progressive Cactus alignment of 577 vertebrate genomes. The alignment was produced with Cactus v3.2.0 from the VGP phase-1 ROADIES tree (v1.1.16b) plus hg38 and mm39; the seqfile, guide tree, and per-reference MAF + PhyloP commands are documented in the source repository at github.com/glennhickey/vgp-cactus.

Methods

Each track was produced with cactus-phast from Cactus v3.2.0. cactus-phast is a Toil-parallelized wrapper around the PHAST package (phyloFit + phyloP) for whole-genome conservation scoring; see the progressive.md documentation for a pipeline walkthrough.

For each reference genome:

  1. The single-coverage MAF anchored at the reference was exported from the HAL with cactus-hal2maf --outType norm single.
  2. 4-fold-degenerate sites were extracted (msa_view --4d) from the reference's CDS annotation, single-cover-filtered, and aggregated across reference contigs.
  3. A neutral substitution model was trained with phyloFit --EM --subst-mod REV --precision HIGH; its background frequencies were then symmetrized with modFreqs to enforce reversibility (mirroring the UCSC published phyloP-track convention).
  4. Per-base conservation scores were computed with phyloP --method LRT --mode CONACC --wig-scores, against the full 577-way tree.

Conservation scores are signed -log10(p): positive values indicate unexpected conservation, negative values indicate unexpected acceleration relative to the neutral model.

Per-genome details

The exact cactus-phast command line used to produce each track is recorded on the corresponding track's description page (click the track name in the browser). The CDS annotation source is also documented per-genome — for GCF_* assemblies it is the NCBI RefSeq GFF for that accession; for hg38, mm39, and hs1 it is the UCSC gene set.

Source data

The HAL alignment, MAFs, trained models (.mod), aggregated 4d-site sufficient statistics (.4d.ss), and these per-base PhyloP tracks are available from the genomeark S3 bucket.
